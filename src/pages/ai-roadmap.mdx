# AI-Destəkli Proqram mühəndisliyi (AI-Assisted Software Engineering) 🤖💻

> **"Süni İntellekt (AI) səni əvəz etməyəcək, amma AI istifadə edən proqramçı səni əvəz edəcək."**

Bu yol xəritəsi sizə ChatGPT, Claude, GitHub Copilot kimi Süni İntellekt alətlərindən istifadə edərək necə 10x daha məhsuldar və effektiv bir proqram mühəndisi ola biləcəyinizi öyrədəcək.

**Səviyyələr:**
- **Level 0:** AI Alətlərinə Baxış (İlk Addımlar) - 1 həftə
- **Level 1:** Prompt Engineering (Düzgün Sual Vermək) - 1-2 həftə
- **Level 2:** AI Pair Programming (AI ilə Cütlükdə Kod Yazmaq) - 2-3 həftə
- **Level 3:** Debugging & Refactoring (Xətaları Tapmaq və Kodu Yaxşılaşdırmaq) - 3-4 həftə
- **Level 4:** Sistem Arxitekturası və Planlama - 1-2 ay
- **Level 5:** Etika, Təhlükəsizlik və Gələcək - Davamlı

# Level 1: Prompt Engineering (Düzgün Sual Vermək) 🗣️

AI sənə ancaq verdiyin məlumata ("prompt") uyğun gələn cavab verəcək. Garbage In, Garbage Out (Pis məlumat ver, pis cavab al).

## 1. Kontekst Hökmdardır (Context is King)

<InfoBox type="error" title="PİS PROMPT ❌">
"React error verir, kömək et" 
</InfoBox>

Heç bir insan bu suala cavab verə bilməz, AI niyə versin?

<InfoBox type="success" title="YAXŞI PROMPT ✅">
"Mən React 18 və Vite istifadə edirəm. `useEffect` daxilində state-i yeniləmək istədikdə 'Too many re-renders' xətası alıram. Kodum belədir: `// kodunuz`. Bunun səbəbi nədir və necə həll edə bilərəm?"
</InfoBox>

**Mükəmməl Promptun 3 Qızıl Qaydası:**
1. **Texnologiya stekini** (versiya daxil) de.
2. **Məqsədini** (nə etmək istədiyini) aşkar izah et.
3. **Mövcud Kodu və ya Error Logunu** yapışdır.

## 2. Zero-Shot vs Few-Shot Prompting
Konteksti artırdıqca cavabın keyfiyyəti geometrik şəkildə artır.

<ConceptCard title="Zero-Shot (Sıfır İpucu)" subtitle="AI-ın insafına qalmaq">
"Mənə istifadəçi siyahısını validasiya edən Python funksiyası yaz." (AI öz bildiyi kimi, sənə aid olmayan bir formatda yazacaq).
</ConceptCard>

<ConceptCard title="Few-Shot (Nümunəvi İpucu)" subtitle="Dəqiq format idarəsi">
AI-a nümunə göstərdikdə kod tərzi sənin layihənin kod tərzinə uyğun olur.
*Nümunə Prompt:* 
"Aşağıdakı kimi bir validasiya funksiyası yaz:
Nümunə: `isEmailValid("test@test.com") // qaytarır: true`
İndi sən eyni strukturda `isPhoneValid(number)` yaz."
</ConceptCard>

## 3. Rol Təyin Etmək (System Prompt / Roleplaying)
AI-a spesifik bir arxetip, vəzifə və ton təyin etdiyinizdə o həmin xarakterin çərçivəsində, cəfəngiyatdan uzaq və daha qatı professional cavablar verir.

<InfoBox type="tip" title="Roleplaying Nümunəsi">
"Sən 20 illik təcrübəsi olan Senior DevOps mühəndisisən və AWS təhlükəsizliyi üzrə eksertsən. Kodumdakı təhlükəsizlik boşluqlarını çox sərt şəkildə kritika et və ən yaxşı enterprise standartlarındakı həllini göstər. Sadəcə kodu ver, başqa heç bir giriş cümləsi qurma."
</InfoBox>

# Level 2: AI as a Pair Programmer 🧑‍💻🤖

## 1. Boilerplate (Təməl) Kodların Yaradılması
Proqramlaşdırma vaxtının çoxu strukturu qurmağa və eyni təkrar kodları yazmağa gedir. Bunu AI-a verin.

<ConceptCard title="Boilerplate Generator" subtitle="Təməl arxitektura">
**Prompt:** "Express.js, TypeScript və PostgreSQL istifadə edərək bir istifadəçi qeydiyyatı (Auth) modulu üçün controller, service və route strukturunu hazırlayıb mənə ver. Şifrələri bcrypt ilə həşlə və JWT qaytar. Tam qovluq strukturunu göstər."
</ConceptCard>

## 2. Unit Testlərin Yazılması
Test yazmaq yorucudur amma enterprise layihələrdə mütləqdir.
1. Kodunuzu AI-a kopyalayın və ya Cursor-da həmin faylı "Context"-ə atın (`@filename`).
2. **Prompt:** "Aşağıdakı bu `calculateTotalDiscount` funksiyası üçün Jest və TypeScript istifadə edərək Unit testlər yaz. Bütün edge-case-ləri (boş array, mənfi məbləğ, tip xətaları) yoxla."
3. AI bir neçə saniyəyə testləri yazacaq. Sənə qalan sadəcə onları oxuyub təsdiqləməkdir.

## 3. RegEx (Regular Expressions) və SQL Sorğuları
RegEx sintaksisini və mürəkkəb SQL cədvəl birləşmələrini (JOIN) əzbərləməyə ehtiyac qalmır.

<InfoBox type="tip" title="SADƏ İZAH İLƏ MÜRƏKKƏB KOD">

# Level 3: Debugging & Refactoring 🐛🔨

## 1. Rubber Duck Debugging 2.0
Əvvəllər proqramçılar kodu plastmas bir ördəkçəyə danışaraq izah edirdilər ki, problemi tapsınlar. İndi "Ağamlı Ördək" var.

<InfoBox type="success" title="Öz Səhvini Özün Tap">
**Prompt:** "Mən bu kodun necə işlədiyini tam anlamıram. Davamlı olaraq state update xətası alıram. Zəhmət olmasa bu React komponentinin render dövrünü (render cycle) və state idarəetməsini sətir-sətir mənə əsas insan dilində xırda detalları ilə izah et."
Çox vaxt AI kodu sənə izah edəndə **sən özün səhvini tapırsan**.
</InfoBox>

## 2. Refactoring (Kodu Təmizləmək və Yaxşılaşdırmaq)
Zibil (spaghetti) kodu götür, mükəmməl koda çevir.

<ConceptCard title="Clean Code Çevirici" subtitle="Senior proqramçı kimi dizayn et">
**Prompt:** "Gördüyün bu JavaScript kodu çox pis ardıcıl İF-ELSE bloku yazıb. Bunu vizual olaraq oxunması çətindirvə maintainable deyil. Bunu Obyektə-yönümlü dizayn pattern-ləri istifadə edərək (məsələn: Map obyekti, Dictionary Look-up və ya Strategy pattern) daha professional və təmiz (Clean Code) versiyaya 'Refactor' et və SOLID prinsiplərinə uyğunlaşdır."
</ConceptCard>

## 3. Legacy (Köhnə) Kodu Oxumaq
İşə yeni girdin, və 7 il əvvəl yazılmış sənədsiz (undocumented) Perl/PHP kodunu anlamaq lazımdır.

<InfoBox type="warning" title="Qorxulu Kod Bazaları">
Əgər şirkət kodunun bir hissəsini anlaya bilmirsənsə, abstraktizasiya tələb et.
**Prompt:** "Mən PHP dilində və qədim Zend fremvorkündə rahat deyiləm, amma bu faylları başa düşməliyəm. Zəhmət olmasa bu faylların yekun məqsədini yüksək səviyyəli bir arxitektura olaraq C# proqramçısının (öz ana dilin) anlaya biləcəyi terminlərlə mənə abstrakt izah et."
</InfoBox>

Gələcəkdə böyüməni (Scale) düşünərək ACID prinsiplərinə necə əməl edəcəyimi də daxil et və qərar verməm üçün mənə fərqli variantlar təklif et."

# Level 5: Etika, Təhlükəsizlik və Hallüsinasiya 🚨

Süni intellektə o qədər də güvənməyin, kor-koranə tətbiq edilən AI həlləri həm məntiqi, həm də fəlakətli təhlükəsizlik problemlərinə gətirib çıxarır.

## 1. Hallucinations (Hallüsinasiya)
AI bəzən ümumiyyətlə mövcud olmayan kitabxanalar (libraries), saxta sənədlər və ya funksiyalar uydurur! Bu ona görə baş verir ki, LLM-lər əslində həqiqəti yox, "ehtimal olunan növbəti sözü" təxmin edirlər.

<InfoBox type="error" title="Kor-Koranə Copy-Paste Etmə!">
**Həll:** Verilən kodları oxumadan (copy-paste edib) layihəyə atmaq olmaz. Həmişə təklif edilən kitabxananın (`npm install fiktiv-paket`) rəsmi dokumentasiyasına bax!
</InfoBox>

## 2. Təhlükəsizlik və Məxfilik (Security / Privacy)

<ConceptCard title="AI-a Nələri ƏSLA Göndərməli Deyilsən" subtitle="Data Məxfiliyi">
1. **API Keys / Secrets:** AWS key, Stripe token, Database şifrələri.
2. **PII (Personal Identifiable Information):** Müştərilərin real adları, email və pasport nömrələri.
3. **Core Intellectual Property (IP):** Şirkətin ən gizli hesablanma alqoritmləri.
</ConceptCard>

**Səbəb:** Public AI modelləri sizin verdiyiniz məlumatlardan öyrənir (training data) və təsadüfən başqa bir istifadəçiyə həmin məlumatı verə bilər! Əgər iş kodunu incələdirsənsə, məlumatı **Anonimləşdir** (Dəyişən adlarını və bazanı `fake_user`, `xxx_token` kimi şərti adlarla dəyişdir). 

## 3. Asılılıq Yaradır (The AI Crutch)
Əgər sadə bir `for loop`-u, təməl string manipulyasiyasını da AI-a yazdırırsansa, psixoloji asılılıq yaranır. Alqoritmik düşüncəni itirməmək üçün hərdənbir AI-ı bağla, öz beyninlə düşün və LeetCode həll et!
