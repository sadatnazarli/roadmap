### 🧠 Deep Dive: Yaddaşda Dəyişənlər

RAM bir böyük rəfli şkaf kimidir, hər rəfin nömrəsi (ünvanı) və içindəki əşya (dəyəri) var.

# Level 2: Orta Səviyyə 🎓

## 2.1 Massivlər (Arrays)

### Bir Ölçülü Massiv:
```c
int numbers[5] = {1, 2, 3, 4, 5};

// Elementlərə giriş
printf("%d\n", numbers[0]); // 1

// Dəyişdirmək
numbers[2] = 10;

// Döngü ilə
for (int i = 0; i < 5; i++) {
    printf("%d ", numbers[i]);
}
```

### Çox Ölçülü Massivlər:
```c
int matrix[3][3] = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

// Giriş
printf("%d\n", matrix[1][2]); // 6

// İç-içə döngü
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        printf("%d ", matrix[i][j]);
    }
    printf("\n");
}
```

### 🧠 Deep Dive: Massivlər Yaddaşda Necə Görünür?
Massivlər yaddaşda **yanaşı** (contiguous) düzülür.
Əgər `int arr[3] = {10, 20, 30}` varsa və `arr` ünvanı `1000`-dirsə:
- `arr[0]` -> Ünvan 1000 (Dəyər: 10)
- `arr[1]` -> Ünvan 1004 (Dəyər: 20) (çünki int 4 baytdır)
- `arr[2]` -> Ünvan 1008 (Dəyər: 30)
Buna görə massivlərə indekslə çatmaq çox sürətlidir (`O(1)`), çünki kompüter sadəcə vurma əməliyyatı edir: `Başlanğıc + (İndeks * Ölçü)`.

### Praktika:
- Massivin ən böyük/kiçik elementi
- Massiv sıralama (bubble sort)
- İki massivin birləşdirilməsi
- Matris əməliyyatları (toplama, vurma)
- Tic-Tac-Toe oyunu

### Pointer Arifmetikası:
```c

> **💡 Pro Tip: Pointer Arithmetic**
> Pointeri artırdıqda (`p++`), o yaddaşda `1 bayt` yox, `sizeof(type)` qədər irəliləyir.
> Məsələn, `int *p` üçün `p+1` əslində yaddaşda 4 bayt irəli gedir. Buna görə massivlərdə gəzmək mümkündür!
```

### Pass by Value vs Pass by Reference:
```c

    x++;  // orijinal dəyişməz

}
```

### 🧠 Deep Dive: Struct Padding (Memory Alignment)
Sən düşünə bilərsən ki, `char` (1 bayt) + `int` (4 bayt) = 5 bayt yer tutur. Amma yaddaşda bu **8 bayt** ola bilər!
Kompüter prosessorları məlumatı 4 və ya 8 baytlıq bloklarla oxumağı sevir.
Buna görə `char`-dan sonra 3 baytlıq "boşluq" (padding) buraxılır ki, növbəti `int` düzgün ünvana düşsün.
Bundan qaçmaq üçün `__attribute__((packed))` istifadə olunur (amma sürət azalır).

## 3.3 Sıralama və Axtarış Alqoritmləri

### Sıralama:
```c
// Bubble Sort
void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        for (int j = 0; j < n-i-1; j++) {
            if (arr[j] > arr[j+1]) {
                int temp = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = temp;
            }
        }
    }
}

// Quick Sort
void quick_sort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quick_sort(arr, low, pi - 1);
        quick_sort(arr, pi + 1, high);
    }
}

// Merge Sort
void merge_sort(int arr[], int l, int r) {
    if (l < r) {
        int m = l + (r - l) / 2;
        merge_sort(arr, l, m);
        merge_sort(arr, m + 1, r);
        merge(arr, l, m, r);
    }
}
```

### Axtarış:
```c
// Linear Search
int linear_search(int arr[], int n, int x) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == x) return i;
    }
    return -1;
}

// Binary Search
int binary_search(int arr[], int l, int r, int x) {
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (arr[mid] == x) return mid;
        if (arr[mid] < x) l = mid + 1;
        else r = mid - 1;
    }
    return -1;
}
```

```c
typedef struct {
    void *memory;
    size_t size;
    size_t used;
} Allocator;

Allocator* allocator_create(size_t size) {
    Allocator *alloc = malloc(sizeof(Allocator));
    alloc->memory = malloc(size);
    alloc->size = size;
    alloc->used = 0;
    return alloc;
}

void* allocator_alloc(Allocator *alloc, size_t size) {
    if (alloc->used + size > alloc->size) {
        return NULL;  // Out of memory
    }
    void *ptr = (char*)alloc->memory + alloc->used;
    alloc->used += size;
    return ptr;
}

    read(client_socket, buffer, 1024); // Müştəridən gələni oxu
    printf("Müştəri dedi: %s\n", buffer);

    send(client_socket, "Salam Müştəri!", 14, 0); // Cavab göndər

    close(server_fd);

// Cache-friendly kod

### 🧠 Deep Dive: System Calls (User vs Kernel Mode)
Sənin yazdığın `printf` və ya `fopen` əslində kitabxana funksiyalarıdır. Onlar təkbaşına diskə ya ekrana yaza bilməz.
Onlar Əməliyyat Sisteminə **System Call** (məs: `write`, `open`) göndərir.
Bu zaman proqram **User Mode**-dan **Kernel Mode**-a keçir. Kernel (nüvə) təhlükəsizlik yoxlanışı aparır, işi görür və nəticəni qaytarır.
Hər system call bahalıdır (vaxt aparır), ona görə buffer (məs: `FILE*`) istifadə olunur.

### Inter-Process Communication:
```c
// Pipes
int pipefd[2];
pipe(pipefd);

if (fork() == 0) {
    close(pipefd[0]);
    write(pipefd[1], "mesaj", 6);
} else {
    char buf[100];
    close(pipefd[1]);
    read(pipefd[0], buf, 100);
}

// Shared Memory
int shmid = shmget(IPC_PRIVATE, 1024, IPC_CREAT | 0666);
char *shared = (char*)shmat(shmid, NULL, 0);
