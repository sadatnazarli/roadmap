### 🧠 Deep Dive: Kernel vs User Space

2.  **Kernel Space**: RAM, CPU və disklə birbaşa danışan "BOSS". Sənin proqramın fayl oxumaq istəyəndə Kernel-ə xahiş edir (**System Call**).

# Level 1: File System & Navigation 📁

## Linux File System Hierarchy

```
/                    Root directory
├── bin/            Essential binaries
├── boot/           Boot loader files
├── dev/            Device files
├── etc/            Config files
├── home/           User directories
├── lib/            System libraries
├── media/          Removable media
├── mnt/            Mount points
├── opt/            Optional software
├── proc/           Process info
├── root/           Root user home
├── sbin/           System binaries
├── tmp/            Temporary files
├── usr/            User programs
└── var/            Variable data
```

## Navigation Komandaları

```bash
# List - faylları siyahıya al
ls              # cari folder
ls -l           # long format (detailed)
ls -la          # hidden fayllar da (. ilə başlayan)
ls -lh          # human-readable size
ls -lt          # tarixi sort
ls -lS          # size-a görə sort
ls -R           # recursive (subfolders da)

# Change Directory
cd /path/to/directory   # absolute path
cd folder               # relative path
cd ..                   # parent folder
cd ~                    # home directory
cd -                    # əvvəlki folder

# Folder strukturunu göstər
tree                   # cari folder
tree -L 2             # 2 səviyyə
tree -d               # yalnız folderləri
```

## File Operations

```bash
# Fayl yarat
touch file.txt
touch {1..10}.txt

# Folder yarat
mkdir folder
mkdir -p path/to/folder

# Kopyala
cp source.txt dest.txt
cp -r folder/ destination/

# Köçür və ya ad dəyişdir
mv old.txt new.txt
mv file.txt /path/to/folder/

# Sil
rm file.txt
rm -r folder/
rm -rf folder/  # force (təhlükəli!)

# Faylı göstər
cat file.txt

# Fayl page-by-page
less file.txt

# Faylın başını/sonunu
head file.txt      # ilk 10 sətir
tail file.txt      # son 10 sətir
tail -f file.txt   # follow (real-time)
```

## File Permissions

```bash
# Permissions (rwxrwxrwx)
chmod 755 file.txt    # rwxr-xr-x
chmod 644 file.txt    # rw-r--r--
chmod 777 file.txt    # rwxrwxrwx (təhlükəli!)
chmod 600 file.txt    # rw-------

# Symbolic mode
chmod u+x file.txt    # user-ə execute
chmod g+w file.txt    # group-a write
chmod o-r file.txt    # other-dən read götür

# Owner dəyişdir
sudo chown user file.txt
sudo chown user:group file.txt
sudo chown -R user:group folder/
```

### 🧠 Deep Dive: Inodes
Sən faylın adını ("file.txt") görürsən, amma Linux onu **Inode Number** (məs: 123456) kimi görür.
Inode faylın metadatasını saxlayır:
*   Sahibi kimdir?
*   İcazələr nədir?
*   Diskdə harada yerləşir?
`ls -i` yazaraq faylların inode nömrəsini görə bilərsən. Faylın adını dəyişsən belə Inode eyni qalır!

```bash
*          # istənilən simvol(lar)

```

# Level 2: Text Processing 📝

## grep - Search in Text

```bash
grep "pattern" file.txt              # axtarış
grep -i "pattern" file.txt           # case-insensitive
grep -r "pattern" folder/            # recursive
grep -n "pattern" file.txt           # line numbers
grep -c "pattern" file.txt           # count
grep -v "pattern" file.txt           # invert match
grep "^start" file.txt               # sətir başı
grep "end$" file.txt                 # sətir sonu
grep "[0-9]" file.txt                # rəqəm olan sətirləri
```

## sed - Stream Editor

```bash
# Əvəzetmə
sed 's/old/new/' file.txt            # ilk occurrence
sed 's/old/new/g' file.txt           # hamısı
sed 's/old/new/gi' file.txt          # case-insensitive
sed -i 's/old/new/g' file.txt        # in-place edit

# Sil
sed '/pattern/d' file.txt            # pattern olan sətirləri
sed '2d' file.txt                    # 2-ci sətri
sed '/^$/d' file.txt                 # boş sətirləri

# Çap
sed -n '2p' file.txt                 # yalnız 2-ci sətri
sed -n '/pattern/p' file.txt         # pattern olanları
```

## awk - Pattern Scanning

```bash
# Column-based processing
awk '{print $1}' file.txt            # 1-ci column
awk '{print $1, $3}' file.txt        # 1-ci və 3-cü
awk '{print $NF}' file.txt           # sonuncu column

# Pattern matching
awk '/pattern/' file.txt             # pattern olan sətirləri
awk '$3 > 100' file.txt              # 3-cü column > 100

# Field separator
awk -F: '{print $1}' /etc/passwd     # : separator
awk -F, '{print $1, $3}' data.csv    # CSV
```

## sort, uniq, cut

```bash
# Sort
sort file.txt                   # əlifba sırası
sort -r file.txt                # tərsinə
sort -n file.txt                # numeric sort
sort -k2 file.txt               # 2-ci column-a görə

# Unique
uniq file.txt                   # ardıcıl duplicate-ləri sil
sort file.txt | uniq            # əvvəl sort et!
sort file.txt | uniq -c         # count ilə

# Cut
cut -d: -f1 /etc/passwd         # : delimiter, 1-ci field
cut -d, -f2 data.csv            # CSV 2-ci column
```

## Pipes & Redirection

```bash
# Pipes (|)
cat file.txt | grep "pattern" | sort | uniq
ls -l | grep ".txt" | wc -l
ps aux | grep "python"

# Output redirection
command > file.txt              # üzərinə yaz
command >> file.txt             # əlavə et
command 2> error.log            # stderr
command &> output.log           # stdout və stderr

# Input redirection
command < input.txt
```

# Level 3: Shell Scripting 🔧

## Bash Script Basics

```bash
#!/bin/bash
# Shebang - hansı interpreter-in istifadə olunacağını göstərir

echo "Salam Dunya!"

# Script-i executable et
chmod +x script.sh

# Run et
./script.sh
```

## Variables

```bash
#!/bin/bash

# Təyin etmə
name="Ali"
age=25
pi=3.14

# İstifadə
echo "Ad: $name"
echo "Ad: ${name}"  # preferred

# Command substitution
current_date=$(date)
files=$(ls)

# Arithmetic
result=$((5 + 3))
result=$((age * 2))
```

## Conditionals

```bash
#!/bin/bash

# if-elif-else
age=20

if [ $age -lt 18 ]; then
    echo "Yetkinlik yaşı deyil"
elif [ $age -lt 65 ]; then
    echo "Adult"
else
    echo "Senior"
fi

# File tests
if [ -f file.txt ]; then     # regular file
    echo "File exists"
fi

if [ -d folder ]; then       # directory
    echo "Folder exists"
fi

# Logical operators
if [ $age -gt 18 ] && [ $age -lt 65 ]; then
    echo "Adult"
fi
```

## Loops

```bash
#!/bin/bash

# for loop
for i in {1..10}; do
    echo $i
done

# C-style
for ((i=1; i<=10; i++)); do
    echo $i
done

# while loop
count=1
while [ $count -le 5 ]; do
    echo "Count: $count"
    count=$((count + 1))
done

# Read file line by line
while read line; do
    echo "Line: $line"
done < file.txt
```

## Functions

```bash
#!/bin/bash

# Sadə funksiya
greet() {
    echo "Salam!"
}

greet  # call

# Parametrli
add() {
    result=$(($1 + $2))
    echo $result
}

sum=$(add 5 3)
echo "Sum: $sum"

# Return value
is_even() {
    if [ $(($1 % 2)) -eq 0 ]; then
        return 0  # true
    else
        return 1  # false
    fi
}

if is_even 4; then
    echo "Even"
else
    echo "Odd"
fi
```

## Arrays

```bash
#!/bin/bash

# Yaratmaq
fruits=("apple" "banana" "orange")
numbers=(1 2 3 4 5)

# Elementlərə giriş
echo ${fruits[0]}       # apple
echo ${fruits[@]}       # hamısı

# Uzunluq
echo ${#fruits[@]}      # 3

# Əlavə et
fruits+=("grape")

# Iterasiya
for fruit in "${fruits[@]}"; do
    echo $fruit
done
```

# Level 4: System Administration 👨‍💼

## Process Management

```bash
# Process Status
ps                      # cari terminal prosesləri
ps aux                  # bütün proseslər
ps aux | grep python    # python prosesləri

# Real-time monitor
top
htop  # better alternative

# Prosesi dayandır
kill PID                # SIGTERM (graceful)
kill -9 PID             # SIGKILL (force)
killall process_name    # ad ilə

# Background/foreground
command &               # background-da işə sal
jobs                    # background jobs
fg %1                   # foreground-a gətir
bg %1                   # background-da davam etdir
```

## System Information

```bash
# OS info
uname -a                # bütün info
uname -r                # kernel version
cat /etc/os-release     # distribution

# CPU info
lscpu                   # CPU info
cat /proc/cpuinfo       # detailed

# Memory info
free -h                 # RAM usage
cat /proc/meminfo       # detailed

# Disk info
df -h                   # disk usage
du -sh folder/          # folder size
lsblk                   # block devices

# Network info
ip addr                 # IP addresses
hostname -I             # IP address
netstat -tuln           # listening ports

# System uptime
uptime                  # uptime və load average
w                       # logged in users
```

## Linux Security & Hardening 🛡️

### SELinux və AppArmor (MacOS/Windows-da olmayan xüsusiyyətlər!)
Bütün Linux prosesləri müəyyən istifadəçi haqqı ilə işləyir. Amma əgər o proses hack edilərsə?
SELinux (RedHat/CentOS) və AppArmor (Ubuntu/Debian) **Mandatory Access Control (MAC)** sistemləridir. Onlar bildirir ki: *"Bu Nginx prosesi ancaq /var/www qovluğunu oxuya bilər, başqa yerə baxa bilməz!"*

```bash
# AppArmor statusu yoxla (Ubuntu)
sudo apparmor_status

# SELinux statusu yoxla (CentOS/RHEL)
sestatus
```

### UFW (Uncomplicated Firewall) və iptables
```bash
# UFW (Asan)
sudo ufw enable
sudo ufw allow ssh          # 22 portuna icazə ver
sudo ufw allow 80/tcp       # HTTP

# iptables (Core Firewall - Dərin öyrənmək istəyənlər üçün)
sudo iptables -L                      # Qaydaları listələ
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT # SSH icazəsi əlavə et
```

### SSH Hardening (Serverin qapısını bərkitmək)
`/etc/ssh/sshd_config` faylını edit edib bu sətirləri dəyiş:
```ini
PermitRootLogin no          # Root girişi bağla
PasswordAuthentication no   # Şifrə ilə girişi bağla (Yalnız SSH Key!)
Port 2222                   # Standart portu dəyiş
```

## Docker və Containerization (Qısa Baxış) 🐳

Docker əslində Linux-un **Namespaces** və **Cgroups** adlı iki vacib kernel texnologiyası üzərində qurulub.
- **Namespaces**: Prosesi izolyasiya edir (sanki başqa maşındadır).
- **Cgroups**: Prosesin neçə CPU və ya nə qədər RAM istifadə edəcəyini limitləyir.

```bash
# Images
docker images                           # list
docker pull ubuntu:20.04                # download
docker build -t name:tag .              # build

# Containers
docker run -it ubuntu bash              # interactive terminal kimi aç
docker run -d -p 8080:80 nginx          # detached mode, port mapping (8080 -> 80)
docker ps                               # işləyən containerləri listələ
docker logs -f container_id             # canlı logları izlə
```
